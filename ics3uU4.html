3. 后端基石：初始化数据库 (init_db)

# 程序启动时调用的第一个核心函数
def init_db():
    # 1. 连接/创建 数据库文件
    con = sqlite3.connect("addressbook.db")
    
    # 2. 召唤游标 (Cursor) 作为SQL执行者
    cur = con.cursor() 

    # 3. 执行建表指令 (多行字符串)
    cur.execute("""
        CREATE TABLE IF NOT EXISTS contacts
        (
            name TEXT,
            tel TEXT
        )
    """)
    
    # 4. 提交更改 (落盘) ！！！极易遗漏！！！
    con.commit()
    
    # 5. 关闭连接，释放文件锁
    con.close()

⚠️
为什么必须写 con.commit()？
SQLite 默认开启事务机制。execute 只是在“内存缓冲区”模拟了建表操作，只有执行 commit() 才会真正将表结构写入硬盘。

5. 核心流程2：安全写入数据库 (占位符机制)

    # 接续 add_data 函数：将校验通过的数据存盘
    con = sqlite3.connect("addressbook.db")
    cur = con.cursor()
    
    # 使用 (?, ?) 问号占位符传参！
    # 绝对不要用字符串拼接(如 f"{name}")防注入
    cur.execute(
        "INSERT INTO contacts VALUES (?, ?)", 
        (name, tel)  # 传入元组 Tuple
    )
    
    con.commit() # 写入硬盘
    con.close()
为什么必须使用 (?, ?) 占位符？
错误示范

💣 字符串拼接 (引发 SQL 注入漏洞)

假设黑客在名字输入框填入：Bob"); DROP TABLE contacts; --

execute(f"INSERT INTO contacts VALUES ('Bob"); DROP TABLE contacts; --', '123')")

后果：原有的 INSERT 被强行截断，并执行了删表命令，数据库崩溃！

正确示范

🛡️ 问号占位符 (Parameterization)

execute("INSERT INTO ... VALUES (?, ?)", (name, tel))

原理：数据库底层将 ? 视为纯数据而非代码，无论输入多危险的字符，都会被当成普通文本强制转义，直接存入表内。

🤔 提前思考：如果是清空 Text (多行) 文本框呢？

由于多行文本框不仅有“列”，还有“行”。所以它不能用数字 0，而是要用类似二维坐标的字符串格式 "1.0"。具体逻辑见下一页详解。

7. 查看全部 (view_all)：拉取数据与 "1.0" 索引揭秘

def view_all():
    con = sqlite3.connect("addressbook.db")
    cur = con.cursor()
    # 获取表中所有数据行
    cur.execute("SELECT * FROM contacts")
    rows = cur.fetchall() # 把数据装进列表
    con.close()

    # 坑点：渲染前必须先清空屏幕历史残留！
    # Text组件专属索引："行号.列号" (行号从1开始)
    text_box.delete("1.0", END)

    if rows:
        for row in rows: # 遍历每一行元组
            text_box.insert(END, 
                "Name: " + row[0] + " Tel: " + row[1] + "\n"
            )
    else:
        text_box.insert(END, "No data found")

为什么要双向绑定？
如果只有(1)，滚轮可用但拖滑块无效；如果只有(2)，拖滑块可用，但打字超过一页时，滑块不会自动变短提示你内容变多。

🎉 Unit 4 知识图谱总结

🖥️ Tkinter 前端三大件

  • 获取输入框文本： entry.get()
  • 清空单行输入框： entry.delete(0, END)
  • 清空多行文本框： text_box.delete("1.0", END)
    （切记：行从 1 算，列从 0 算）
  • 按钮事件绑定： Button(..., command=函数名)
    （千万别写成函数名()，加括号页面一开就自动执行了！）

🗄️ SQLite 后端四步曲

  • 建立连接通道： connect("xx.db") & cursor()
  • 增删改查动作： cursor.execute(SQL语句)
  • 防注入终极杀器： 强迫自己永远使用 (?, ?) 传参。
  • 数据落盘 / 内存：
    写入数据必须 con.commit() 才能存入硬盘。
    查询数据必须 cur.fetchall() 才能拿进内存。
